Ads Player

A React video player with Google IMA SDK integration for pre-roll, mid-roll, and post-roll ad support.

03 — useVideoPlayer Hook

Step-by-Step Execution

What happens from component mount to video playing.

1
Component Mounts
The VideoPlayer component renders a <video> element to the DOM. The hook receives src, adTagUrl, adContainerId, and poster as inputs. State starts as ready: false, error: null.
2
Load Google IMA SDK
loadImaSdk() injects a <script> tag pointing to Google's CDN. The promise is cached — if another component calls it simultaneously, it reuses the same request. A 10-second timeout ensures it never hangs forever (e.g. if an ad-blocker silently drops the request).
https://imasdk.googleapis.com/js/sdkloader/ima3.js
✓ Already loaded → resolve immediately
✓ Script loads → resolve
✗ Network error → reject
✗ 10 seconds pass → reject with timeout error
3
Initialize Video.js Player
Once the IMA SDK is loaded, videojs() is called on the <video> element. It configures the player with the video source, poster image, controls, and fluid (responsive) layout. The MIME type is automatically detected from the file extension (mp4, m3u8, webm, mpd).
sources: [{ src, type: getVideoMimeType(src) }]
.mp4 → video/mp4
.m3u8 → application/x-mpegURL (HLS)
.mpd → application/dash+xml (DASH)
.webm → video/webm
4
Configure IMA Ad Plugin
player.ima() connects the video.js player to the IMA SDK. It receives the ad tag URL (a VMAP/VAST URL from Google Ad Manager) and the ad container ID (the DOM element where ads will be rendered visually).
adTagUrl → URL to VMAP/VAST ad schedule (from Google Ad Manager)
adContainerId → DOM element for rendering the ad overlay
5
Wait for First Play
Ads are not requested immediately — they are requested only when the user first presses Play. This is required by the IMA SDK: initializeAdDisplayContainer() must be called from a user gesture (a click), otherwise browsers block it.
player.on("play", () => {
  player.ima.initializeAdDisplayContainer()
  player.ima.requestAds() ← fetches the ad from Google
})
6
Ad Error Recovery
If an ad fails (network issue, invalid tag, ad-blocker), the adserror event fires. The hook automatically calls resumeAdPlayback() so the video content continues playing even if the ad failed — the user never gets stuck.
7
Ready
ready: true is set. The component removes the loading overlay. The player is fully operational.
04 — Player States

What the User Sees

The component moves through three visual states.

⏳
Loading
ready: false
error: null
→
▶️
Ready
ready: true
error: null
→
⚠️
Error
ready: false
error: "message"


Loading State

Shown immediately on mount while the IMA SDK loads and video.js initializes. A black overlay with "Loading…" covers the player area. The video element is already in the DOM so initialization can proceed.

Ready State

Video.js and the IMA plugin are fully initialized. The loading overlay is removed. The user can press play — on first play the ad is fetched and played, then the video content follows.

Error State

Triggered if the IMA SDK fails to load (timeout, network error) or if video.js fails to initialize. The entire player is replaced with a black 16:9 box and a user-friendly error message.

Ad Error (auto-recovered)

If an individual ad request fails mid-session, it does NOT enter the Error state. The hook silently calls resumeAdPlayback() and the video continues. Transparent to the user.

06 — DOM Structure

How the Player is Laid Out

Three layers stacked on top of each other using CSS absolute positioning.

<div> ← outer wrapper · relative · pt-[56.25%] = 16:9 aspect ratio reserved
<div id="ad-container"> ← absolute inset-0 · z-10 · pointer-events-none
IMA SDK injects ad content here ← skip button, click overlay, etc.
<div data-vjs-player> ← absolute inset-0 · video.js attaches here
<video> ← actual video element · w-full h-full

Why pt-[56.25%]?

56.25% = 9 ÷ 16 × 100. This CSS trick reserves the correct 16:9 height based on width before JavaScript loads, preventing the layout shift/glitch on page load where the player briefly appears at the wrong size.

Why pointer-events-none?

The ad container sits on top of the video. Without this, it would block all clicks (pause, seek bar, volume) when no ad is active. The IMA SDK overrides this on its own child elements when an ad needs to be interactive.
